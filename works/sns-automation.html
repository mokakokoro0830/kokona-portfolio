SNS投稿自動生成AI × GAS × Claude API

カテゴリ

SNS運用 / AI自動化デモ

技術

HTML / CSS / GAS / Claude API / Google Sheets

デプロイ

GitHub Pages

ステータス

LIVE

概要

「毎日のSNS投稿文を考える時間をゼロにしたい」という実際のクラウドワークス案件からヒントを得て制作したデモです。ブランド情報（名前・商品・ターゲット・トーン）を入力するだけで、Claude APIがThreads・Instagram・TikTok向けの投稿文をSNSごとの特性に合わせて自動生成します。

生成された投稿文はGoogleスプレッドシートの「投稿カレンダー」シートに自動保存され、そのまま運用ベースとして活用できます。LP制作・Claude API連携・GASバックエンドをひとつのデモで実証しています。

自動化フロー

  • フォーム入力 — ブランド名・商品/サービス・ターゲット・トーン・追加メモを入力。Threads / Instagram / TikTokから対象SNSを選択し、生成件数（1〜5件）をスライダーで指定
  • GAS（doPost）受信 — WebAppとしてデプロイしたGASがPOSTを受け取り、選択されたSNSごとにループ処理
  • Claude API呼び出し — SNSごとのトーン定義（Threads：500字以内・親しみやすい / Instagram：150字・絵文字活用 / TikTok：100字・若者言葉）をプロンプトに組み込み、claude-opus-4-5で投稿文を生成
  • Sheetsへ保存 — 生成された投稿文・ハッシュタグ・ブランド情報・ステータス（未投稿）をGoogleスプレッドシートに自動追記。ヘッダーは紫背景で自動スタイリング
  • 結果表示 — フォームがSNS別の投稿カードに切り替わり、スプレッドシートへのリンクも表示

技術構成

  • HTML / CSS / JS — フロントエンド（フレームワークなし）。ラベンダー×ピンクのグラデーションUI
  • Google Apps Script — バックエンド（WebApp / doPost）。SNSごとにClaude APIを呼び出すループ処理
  • Claude API（claude-opus-4-5）— SNS別トーン定義をシステムプロンプトに組み込み、JSON配列形式で投稿文を返す
  • SpreadsheetApp — 投稿カレンダーシートへの自動追記・ヘッダースタイリング・列幅調整
  • GitHub Pages — フロントのホスティング

こだわり

  • Threads・Instagram・TikTokそれぞれのプラットフォーム特性をプロンプトに反映。文字数・絵文字・ハッシュタグ数が自動で変わる
  • SNS選択カードはチェックボックスをカード全体でトグル可能にし、直感的なUI操作を実現
  • 生成件数をスライダー（1〜5件）で指定でき、複数投稿を一括生成できる設計
  • GAS側でJSON抽出をregexで行い、Claudeの返答にMarkdownコードブロックが混入しても正常にパースできるよう対処
  • エンドツーエンドのテスト（フォーム送信 → Claude生成確認 → シート記録確認）で動作確認済み
